Step 0

Pre-Demo Setup

Complete these steps before starting the live demo.

1
Open the live site

Navigate to rev-shield.vercel.app in Chrome. Enter the password when prompted.

2
Reset the UI

Refresh the page so the pipeline is in its initial state (no results showing). The "Demo" tab should be selected by default in the Data Source panel.

3
Collapse the Data Source panel

Click the panel header to collapse it — this gives a cleaner starting view with the pipeline stages and Analyze button visible.

4
Enable sound

Make sure the Sound button (top right) is enabled. The analysis pipeline has audio cues that make the demo more engaging.

5
Have a second tab ready

Open the slide deck at rev-shield.vercel.app/demo.html in a separate tab for the intro slides if needed.

Tip: The demo runs best on a large screen. If presenting on a projector, zoom to 110-125% for better visibility of the dashboard metrics.

Part 3

Technical Architecture

Quick reference for Q&A from judges.

Stack

LayerTechnology
FrontendNext.js 16 (App Router), React 19, Tailwind CSS 4, Framer Motion, Recharts
BackendNext.js API Routes (Server-Sent Events for streaming pipeline)
AI / LLMClaude API (narrative generation), Minds AI (conversational agent)
Data SourcesShopify Admin API, Stripe API, Google Sheets, CSV parser, Apify Cheerio Scraper
HostingVercel (production), Vercel Cron Jobs (daily order generation)
AuthCookie-based password protection (AuthGate component)

Key Algorithms

AlgorithmPurpose
RRAF ScoringWeighted composite (Risk 30%, Root Cause 20%, Revenue 35%, Frequency 15%) for prioritizing incidents
LIFT DiagnosisCategorizes root cause into Landing, Incentive, Friction, Trust with signal correlation
Baseline DetectionStatistical comparison of observed vs. expected conversion rates across segments
Agent ActionsRisk-tiered action proposals with auto-approve for low-risk, human approval for medium/high
